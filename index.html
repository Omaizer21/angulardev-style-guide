Architecture Overview

Understanding the core / shared / features pattern for scalable Angular apps.

🧠

core/

Singleton services, guards, interceptors, and utilities used app-wide. Imported only once in the root module. Contains authentication logic, HTTP interceptors, global services, constants, enums, and models.

  • Guards (auth, role, no-auth)
  • Interceptors (auth, error, loading, cache)
  • Singleton services
  • Global models, enums, constants
  • Utility functions
🔗

shared/

Reusable components, directives, and pipes shared across features. Imported by any feature module that needs them. Contains UI building blocks like buttons, modals, tables, and form validators.

  • Reusable UI components
  • Custom directives
  • Custom pipes
  • Form validators
🧩

features/

Lazy-loaded feature modules, each representing a distinct business domain. Each feature has its own pages, components, services, models, and routes — fully self-contained.

  • Auth (login, register, forgot-password)
  • Dashboard (stats, activity feed)
  • Users (list, detail)
  • Each feature has its own routes file
🖼️

layout/

Layout components that define the visual shell of the application — header, sidebar, footer, and the main layout wrapper that composes them together.

  • Header with navigation
  • Sidebar navigation
  • Footer
  • Main layout composition

Naming Conventions

Angular's consistent naming conventions for files, classes, and selectors.

.component.ts

Components

UI building blocks. File: hero.component.ts → Class: HeroComponent → Selector: app-hero

.service.ts

Services

Business logic & data access. File: auth.service.ts → Class: AuthService

.guard.ts

Guards

Route protection. File: auth.guard.ts → Function: authGuard

.interceptor.ts

Interceptors

HTTP request/response middleware. File: auth.interceptor.ts → Function: authInterceptor

.pipe.ts

Pipes

Template data transformation. File: truncate.pipe.ts → Class: TruncatePipe

.directive.ts

Directives

DOM behavior modifiers. File: highlight.directive.ts → Class: HighlightDirective

.model.ts

Models

TypeScript interfaces/types. File: user.model.ts → Interface: User

.spec.ts

Tests

Unit tests. File: hero.component.spec.ts → Co-located with source file

.routes.ts

Routes

Route definitions. File: auth.routes.ts → Export: AUTH_ROUTES

.enum.ts

Enums

Enumerated types. File: role.enum.ts → Enum: Role

.constant.ts

Constants

App-wide constants. File: api-endpoints.constant.ts → Export: API_ENDPOINTS

.util.ts

Utilities

Helper functions. File: date.util.ts → Pure functions

Key Principles

The design principles behind a well-architected Angular application.

S

Single Responsibility

Each file, class, and module should have one reason to change. Components handle presentation; services handle data and logic.

O

Open / Closed

Design modules and services to be extended without modifying existing code. Use interfaces and dependency injection.

L

Liskov Substitution

Derived classes or implementations should be substitutable for their base types without breaking the app.

I

Interface Segregation

Prefer many small, specific interfaces over one large interface. Keep service contracts lean and focused.

D

Dependency Inversion

Depend on abstractions, not concretions. Angular's DI system naturally encourages this via injection tokens and interfaces.

🔄 DRY — Don't Repeat Yourself

Extract shared logic into services, pipes, directives, and utility functions. The shared/ folder is your best friend.

📦 Lazy Loading

Each feature module should be lazy-loaded via loadChildren or loadComponent in routes to keep the initial bundle small.

🗂️ Feature Encapsulation

Features own their pages, components, services, and models. Only the routes file is exposed to the rest of the app.

🧪 Co-located Tests

Spec files live next to the code they test (*.spec.ts). This makes it easy to find and maintain tests.

Quick Reference Cheat Sheet

Essential Angular CLI commands and quick tips.

🚀 Generate Components

ng g c features/auth/pages/login
ng g c shared/components/button
ng g c layout/header

⚙️ Generate Services

ng g s core/services/auth
ng g s features/users/services/users-api

🛡️ Generate Guards

ng g guard core/guards/auth
ng g guard core/guards/role

🔌 Generate Interceptors

ng g interceptor core/interceptors/auth
ng g interceptor core/interceptors/error

🔧 Generate Pipes

ng g pipe shared/pipes/truncate
ng g pipe shared/pipes/time-ago

📌 Generate Directives

ng g directive shared/directives/highlight
ng g directive shared/directives/tooltip

📁 Recommended Structure

src/app/
├── core/       ← singletons
├── shared/     ← reusables
├── features/   ← lazy modules
└── layout/     ← shell UI

💡 Tips

  • Use standalone components (Angular 15+)
  • Barrel exports via index.ts
  • Environment configs for API URLs
  • Prefix components: app-
